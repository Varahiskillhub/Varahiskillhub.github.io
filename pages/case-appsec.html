AppSec testing + developer remediation partnership

How I run web/API testing and drive issues to closure in a way engineers actually like.

Problem

Findings that are hard to reproduce or lack context slow down remediation.

Approach

  • Test with focus on authn/authz, access control, session, input handling.
  • Write developer-friendly reports: repro steps, impact, screenshots/requests, fix guidance.
  • Prioritise by exploitability and business impact.
  • Re-test fixes and verify closure (including edge cases).

Outcome

  • Faster remediation cycles and fewer repeat issues.
  • Clearer risk communication for stakeholders.

Template I use

  • Summary → Impact → Steps to reproduce → Evidence → Root cause → Fix guidance → Verification
